This directory contains the main quality assurance documentation produced during the project.

The documents support the testing process by organizing planning, execution, analysis, and innovation artifacts, providing visibility into test strategy, coverage, results, and added value.

### Structure

- **mindmaps/** – Visual artifacts used to explore application flows, risks, and test scenarios.
- **test-plans/** – Test planning documents defining scope, approach, risks, and test strategy.
- **report/** – Test execution reports, evidence, and analysis of findings and defects.
- **innovation-part/** – Documentation explaining how the innovation component of the project was conceived, developed, and integrated.
